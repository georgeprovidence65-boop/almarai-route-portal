# Almarai Route Portal - Multi-Language Requirement

## Correct Requirement

The app should open in the language used by the user's phone or computer whenever possible. It should also provide a language selector in settings or on the page so the user can manually choose another language.

## Required Languages

- English
- Arabic
- Hindi
- Sinhala, for Sri Lanka
- Bangla, for Bangladesh
- Urdu, for Pakistan

## How The App Should Choose Language

1. First, use the language the user selected inside the app.
2. If the user has not selected a language, use the phone/computer browser language.
3. If the browser language is not supported, use English.

## Language Selector

Every main portal should have a clear language selector. It can appear in the header or settings area.

Suggested options:

- English
- Arabic
- Hindi
- Sinhala
- Bangla
- Urdu

When a user changes language, the choice should be saved on that device and reused the next time they open the app.

## Direction Rules

- Arabic and Urdu should use right-to-left layout direction.
- English, Hindi, Sinhala, and Bangla should use left-to-right layout direction.

## Important Pages To Translate

- Company QR page
- Login page
- Manager portal
- Admin dashboard
- Salesman dashboard
- Customer portal

## Business Terms That Must Stay Clear

- `C - Cash`
- `CD - Credit`
- Route
- Area
- Customer
- Product
- Price
- Stock
- Transfer
- Request Help
- Offer Help
- Depot Support
- Manager
- Admin
- Salesman

## Recommended Rollout

Add the language system in stages:

1. Add automatic language detection and language selector.
2. Translate login and QR pages first because every user sees them.
3. Translate customer portal.
4. Translate salesman portal.
5. Translate manager and admin portals.
